# Security Policy

We take the security of CPKit seriously. If you believe you have found a security vulnerability, please report it to us responsibly using the procedure outlined below.

---

## Supported Versions

We actively support and fix security issues on the following versions:

| Version | Supported |
| :--- | :--- |
| `v0.1.x` | 🟢 Yes |
| `< v0.1` | 🔴 No |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities publicly using GitHub Issues.**

Instead, please report security issues privately by emailing the project maintainer, MD. Masudul Hasan, at `23240442@sstu.ac.bd`.

Please include the following details in your report:
1. **Description**: Detail the vulnerability and potential impact.
2. **Steps to Reproduce**: Provide clear instructions or a proof-of-concept (PoC) script.
3. **Environment**: Mention your browser, operating system, and CPKit version.

---

## Our Response Process

Once a vulnerability report is received:
1. We will acknowledge receipt of your email within **48 hours**.
2. We will investigate the issue and coordinate a fix.
3. We will keep you updated on our progress as we implement a resolution.
4. Once resolved, we will publish a patch release (e.g. `v0.1.1`) and credit you for the discovery, unless you prefer to remain anonymous.
