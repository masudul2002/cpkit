# Support Guide

Welcome to the CPKit support page. Before opening a support request, please review this guide to find the quickest path to resolving your issue.

---

## 🔍 Step 1: Self-Help Resources

Before opening a new ticket, check if your question has already been answered:
- **Design System Guides**: Check our [Design System Documentation](docs/design-system.md) for UI layout structure instructions.
- **Troubleshooting**: Search existing [GitHub Issues](https://github.com) or [Discussions](https://github.com) to see if another user faced a similar problem.

---

## 🛠️ Step 2: Getting Support

If you still need help, please select the appropriate channel below:

### 1. Reporting Bugs
If you found a functional error (e.g. compiler outputs not copying, search dialog freezing, layout overflows):
- Open a [GitHub Issue](https://github.com).
- Choose the **Bug Report** template.
- Provide step-by-step reproduction instructions.

### 2. Requesting Features
If you want to suggest a new CP reference library, stress test framework, or visualization widget:
- Open a [GitHub Issue](https://github.com).
- Choose the **Feature Request** template.
- Explain the benefit of the tool for competitive programmers.

### 3. Asking Questions & General Help
If you need help configuring templates, building the project, or understanding algorithm references:
- Open a **Question** issue template, or post in the **Discussions** page.
